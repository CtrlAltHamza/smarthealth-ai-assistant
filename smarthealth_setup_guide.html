SmartHealth — Local Setup Guide
VS Code + Backend + GitHub + Jira · Step by step
1
Install prerequisites on your PC
Node.js 20 LTS
required ▶
Download from nodejs.org → choose 20 LTS. Installs node and npm together.
node -v # should show v20.x.x npm -v # should show 10.x.x
Python 3.11
required (AI service) ▶
Download from python.org. During install: ✅ check "Add Python to PATH".
python --version # should show 3.11.x
On Windows, you may need to use python3 instead of python.
PostgreSQL 15
required (database) ▶
Download from postgresql.org. During install, set password to postgres123 (matches the .env).
# After install, open pgAdmin or psql and create DB: CREATE DATABASE smarthealth;
pgAdmin is a visual tool that installs alongside PostgreSQL — you can use it to view tables and run queries.
Git
required ▶
Download from git-scm.com. Accept all defaults during install.
git --version
VS Code + extensions
required ▶
Download from code.visualstudio.com. Then install these extensions:
ESLint Prettier Python (Microsoft) REST Client GitLens Thunder Client ES7+ React Snippets
Open VS Code → press Ctrl+Shift+X to open Extensions panel → search each name above.
Docker Desktop (optional but recommended)
optional ▶
Download from docker.com. Lets you run all 5 services with one command instead of manually starting each one.
# One-command start (with Docker): docker compose up --build
2
Open & configure the project in VS Code
Unzip and open in VS Code
first thing ▶
Extract the zip you downloaded. Then in VS Code: File → Open Folder → select the smarthealth folder.
# Or from terminal: cd smarthealth code .
You should see frontend/, backend/, ai-service/ in the file explorer sidebar.
Create the .env file
required ▶
Copy the example env file and edit it:
# In the project root: cp .env.example .env
Open .env and make sure DB_PASSWORD matches what you set in PostgreSQL. Leave everything else as-is for local dev.
3
Run the backend (Node.js)
Install & start the backend
terminal 1 ▶
Open a terminal in VS Code (Ctrl+`), then:
cd backend npm install # installs all packages (~2 min) npm run dev # starts server on :5000
You should see: "Database connected successfully" and "Server running on port 5000". If you see a DB error, check PostgreSQL is running and your .env password matches.
# Test it's working: # Open browser → http://localhost:5000/api/health # Should show: {"status":"OK",...} # Swagger docs → http://localhost:5000/api/docs
Run DB migrations (first time only)
first time only ▶
This creates all the database tables:
cd backend npx sequelize-cli db:migrate # creates tables npx sequelize-cli db:seed:all # adds demo data (optional)
The demo seed creates the 3 test accounts: patient@demo.com, doctor@demo.com, admin@demo.com — all with password Demo1234!
Run the frontend
terminal 2 ▶
Open a new terminal tab (click + in VS Code terminal):
cd frontend npm install # ~3 min first time npm start # opens http://localhost:3000 in browser
Run the AI service (Python)
terminal 3 ▶
Open a third terminal tab:
cd ai-service python -m venv venv # create virtual env venv\Scripts\activate # Windows (use source venv/bin/activate on Mac) pip install -r requirements.txt # installs FastAPI, spaCy etc. python -m spacy download en_core_web_sm # downloads NLP model uvicorn main:app --reload --port 8000 # starts AI on :8000
AI docs at: http://localhost:8000/docs. The app works without the AI service running, but the symptom checker won't work.
4
Kaggle / Google Colab for AI model training
Use Kaggle for dataset + training (recommended)
free GPU ▶
The AI model needs a symptom-disease dataset. Kaggle has exactly this — free to use with free GPU.
Step 1 — Get dataset Go to kaggle.com → "Disease Symptom Dataset". Download dataset.csv.
Step 2 — Train on Kaggle Create a new Notebook on Kaggle, upload the CSV, run the training code (I'll generate this for you as Sprint 4).
Kaggle gives you 30h/week free GPU. Training the Random Forest model takes ~5 minutes. Download the resulting rf_model.pkl and put it in ai-service/models/.
Or use Google Colab (alternative)
free ▶
Open colab.research.google.com → New notebook. Same process — upload dataset, train, download .pkl.
Colab = better for quick experiments. Kaggle = better for structured notebooks that stay saved and shareable with supervisor.
5
GitHub setup & commits
Create repo & push initial code
one-time setup ▶
Go to github.com/new → name it smarthealth-ai-assistant → Private → Create. Then:
cd smarthealth git init git add . git commit -m "feat: Sprint 1 - project scaffold, backend API, AI service, frontend shell" git branch -M main git remote add origin https://github.com/YOUR_USERNAME/smarthealth-ai-assistant.git git push -u origin main
Create branches for each sprint
team workflow ▶
Each team member works on their own branch, then merges via Pull Request:
# Create a branch for the feature you're working on: git checkout -b feature/SHAI-12-auth-login # After making changes: git add . git commit -m "feat(auth): add JWT login endpoint SHAI-12" git push origin feature/SHAI-12-auth-login # Then open a Pull Request on GitHub to merge into main
main branch = production develop branch = integration feature/* = your work
Commit message convention (for Jira linking)
important for marks ▶
Include the Jira ticket number in every commit — this auto-links commits to tickets:
git commit -m "feat(auth): implement JWT register endpoint SHAI-5" git commit -m "fix(db): fix patient model FK constraint SHAI-8" git commit -m "feat(ai): add symptom NLP pipeline SHAI-15" git commit -m "test(auth): add login unit tests SHAI-6" git commit -m "docs: update README with setup guide SHAI-3"
Format: type(scope): message TICKET-ID. Types: feat, fix, test, docs, chore, refactor. This creates a full audit trail your supervisor can see.
6
Jira setup (Scrum board)
Create project & configure board
free plan ▶
Go to atlassian.com/jira/free → sign up free (up to 10 users).
Project settings: Project type: Scrum Project name: SmartHealth AI Assistant Project key: SHAI Board name: Sprint Board
Workflow statuses Backlog → Selected for Dev → In Progress → Code Review → Testing → Blocked → Ready for Deploy → Done
Create 3 team members Invite your teammates by email. Assign roles: Backend Lead, Frontend Lead, AI/ML Lead.
Create Epics
do this first ▶
In Jira: Backlog → Create Epic for each module. Use these exact names:
SHAI-E1 Project Setup SHAI-E2 Authentication SHAI-E3 Frontend UI SHAI-E4 Backend APIs SHAI-E5 Database SHAI-E6 AI Symptom Analysis SHAI-E7 Disease Prediction SHAI-E8 Appointments SHAI-E9 Dashboard SHAI-E10 Testing & QA SHAI-E11 DevOps SHAI-E12 Documentation
Create Sprint 1 stories & start
sprint kickoff ▶
Create a Sprint called "Sprint 1 — Setup & Architecture". Add these user stories to it:
User stories for Sprint 1 (copy these into Jira): SHAI-1 As a dev, I can clone the repo and run the project locally (3 pts) SHAI-2 As a dev, Docker Compose starts all services in one command (5 pts) SHAI-3 GitHub repo created with branch strategy and CI/CD configured (3 pts) SHAI-4 Database schema designed and all Sequelize models created (8 pts) SHAI-5 As a patient, I can register with email and password (5 pts) SHAI-6 As a user, I can login and receive a JWT token (5 pts) SHAI-7 API documentation available at /api/docs via Swagger (3 pts)
Click "Start Sprint" → set 2-week duration. Move each story through the workflow as you work on them. Your supervisor will see the burndown chart update in real time.
Link GitHub to Jira
bonus marks ▶
In Jira: Project Settings → GitHub → Connect Repository. Once linked, any commit message containing SHAI-5 auto-appears inside that Jira ticket — your supervisor sees the exact code change tied to the story.
This is what makes your project look genuinely professional. It creates a full traceability chain: requirement → story → commit → test.
